# Product

## Register

product

## Users

需要管理本地或企业知识资料的程序员、知识工作者与 Agent 平台维护者。他们在控制台中持续上传、编译、检查、搜索和维护 Wiki，希望快速完成任务并清楚掌握知识库状态。

## Product Purpose

KnowLLM 将原始资料编译为可编辑、可检索、可审计的 Markdown Wiki，并作为人和 Agent 共用的长期知识工作区。成功的产品体验应让资料导入、Wiki 维护、问题诊断和 Agent 调试都保持透明、直接、可控。

## Brand Personality

专业、克制、高效。界面应像可信赖的工程工作台，强调清晰状态、明确操作和稳定反馈。

## Anti-references

避免信息展示型首页、大块说明文案、装饰性概览卡片、低信息密度、无意义注解、过度阴影和为视觉效果牺牲操作效率的设计。

## Design Principles

- 任务优先：页面首先服务上传、编译、搜索、检查和维护。
- 状态透明：关键状态、错误和操作反馈应容易识别。
- 高效密度：充分利用工作区空间，同时保持清晰层级。
- 熟悉可控：使用标准控件和可预测交互，不隐藏关键后果。
- 人机共用：内容和来源始终可被用户查看、编辑和核验。

## Accessibility & Inclusion

保证清晰文字对比度、可见键盘焦点、标准控件语义、键盘可操作菜单，并尊重系统减少动态效果设置。
