## Design Context

### Users

KnowLLM 的技术操作者和知识工程开发者。他们在桌面端上传不可变 Source、确认编译预算、审阅共享 Staging，并一次性发布整套 Wiki；高频任务是列表查找、批量编译和页面回读。

### Brand Personality

克制、清晰、可靠。界面应像工程工作台：状态可见、操作直接、风险明确，不把尚未由服务端支持的能力伪装成入口。

### Aesthetic Direction

延续现有 LLM Wiki 的浅色紧凑工作台：Geist 字体、中性蓝灰背景、低装饰表格、顶部操作条和 sticky 表头。避免 Hero、数据卡瀑布、重阴影和与任务无关的图形；窄屏时让表格局部滚动、审阅区上下排列。

### Implementation Defaults

已检测到 React 19 + Vite + TypeScript，Tailwind CSS 4，shadcn `radix-nova` 配置和 Lucide 图标。当前项目是 Radix 风格的本地 shadcn primitive 组合；表格使用原生 table 和本地 Pagination，没有引入表单库、数据表格库或虚拟列表。本次保留这些现有基线，不新增 UI 框架或 block accelerator。

### Design Principles

1. 先用排版、间距和状态层级建立可读性，再使用有限的蓝色强调主操作。
2. 顶部只保留全局配置与高频动作，内容区以列表和审阅为中心。
3. 一个界面只呈现服务端已有合同；无后端能力不放禁用占位入口。
4. 破坏性发布和撤销必须清楚说明影响并要求明确确认。
5. 多选、搜索和详情按需展开，默认保持列表工作台简洁。
